Einführung in das Codieren mit 5Code

In diesem Dokument lernen Sie die Grundlagen kennen, wie Sie mit 5Code Java-Code erstellen und bearbeiten. 

  1. Zunächst wird kurz der Unterschied zwischen Codieren und Programieren geklärt.
  2. Auch werden kurz die besonderen Eigenschaften von 5Code für Anfänger dargestellt.
  3. Dann werden die Grundzüge der Codierung in 5Code erläutert und gleich interaktiv geübt.

Einstellungen zu Beginn

Damit 5Code Ihren Code auch compilieren und laufen lassen kann, muss es wissen, in welchem Ordner (unter welchem Pfad) es den Java-Compiler javac und die Virtuelle Maschine java findet. Diese Programme werden dann im Hintergrund von 5Code aufgerufen, wenn es nötig ist.

Das haben Sie beim ersten Start von 5Code schon erledigt, wenn Sie dem Begrüßungs-Dialog gefolgt sind. 

Falls Sie die Einrichtung von Java noch machen müssen, finden Sie hier die Anleitung 
Sie finden die Einrichtung im Menü Projekt -> Einstellungen -> Java-Einstellungen

Im so aufgerufenen Dialog öffnen Sie mit dem Buttoneinen Standard-File-Open-Dialog Ihres Betriebssystems. Navigieren Sie in diesem Dialog zurDateijavac.exe(bzw. zur entsprechenden Datei auf MacOS und Linux) und wählen diese aus. Damit legen Sie den korrekten Pfad fest. Das sieht dann z.B. so aus:

Diesen Pfad müssen Sie nur 1x einstellen.

Außerdem:

Auch bei Aktualisierungen von 5Code oder Java brauchen Sie nichts mehr einstellen
Weder bei Aktualisierungen von 5Code, noch von Java müssen sie die Java-Einstellungen neu einstellen.
5Code wird voraussichtlich im Laufe des Semesters ein paar mal aktualisiert. Wenn es eine neue Version gibt, bekommen Sie eine Nachricht beim Öffnen von 5Code. Wenn Sie die Aktualisierung anstoßen, läuft sie automatisch ab. Danach können Sie ganz normal weiter arbeiten.
 

Java wird ca. alle 6 Monate aktualisiert. Für die Veranstaltung und die Inhalte von OOP1 (und auch OOP2) sind diese Änderungen nicht relevant. Sie brauchen Java (d.h. das JDK) also nicht zu aktualisieren. 

Codieren vs. Programmieren

Unter Programmieren versteht man den kompletten Weg vom Problem zum Programm

  1. Anforderungen lesen
  2. und verstehen
  3. Lösungen überlegen
  4. und aufschreiben
  5. und dann erst codieren

Das Codieren betrifft nur den letzten Teil:

  • Die Umsetzung des Lösungskonzepts in Code in einer Programmiersprache

In der Vorlesung erlernen wir alle Schritte des Programmierens. Anderes als übliche Entwicklungsumgebungen unterstützt Sie 5Code dabei in allen Schritten.

Spezielle Unterstützung von Programmier-Anfängern

5Code ist eine integrierte Entwicklungsumgebung (englisch abgekürzt: "IDE"), 
ähnlich wie eclipse, Visual Studio Code oder IntelliJ IDEA.

Allerdings bietet 5Code besonders Programmier-Anfängern einige Features, 
die das Erlernen des Programmierens besonders unterstützen. 

Insbesondere ist das die Integration der oben beschriebenen Schritte 1.-5. und der einfache Zugang zum Codieren:

  • Der Aufgabentext und die eigenen Lösungen sind immer zusammen verfügbar
    • Sogar in ein und dem gleichen Dokument.
  • Code kann direkt ausprobiert werden
    • ohne sonst notwendigen zusätzlichen Rahmen-Code.
  • Eigene Überlegungen und Lösungsideen können mit dem eigenen Code zusammen geschrieben werden
    • in ein und dem gleichen Dokument. 

Damit haben Sie immer Aufgaben und Ihre eigenen Lösungen beieinander.

Text und Code können in einem Dokument integriert werden

Formatierter Text ist immer übersichtlicher als reiner Programmcode. 5Code ermöglicht es, formatierten Text und Code im gleichen Dokument zu mischen. So ist die übersichtliche Beschreibung sowohl der Aufgabe als auch Ihrer Lösung immer direkt mit dem Lösungs-Code in einem Dokument gespeichert und so zusammen verfügbar. 

Das hilft Ihnen vor allem dann, wenn Sie nicht in mehreren Dokumenten den Zusammenhang zwischen Aufgabe und Lösung suchen müssen. Sie müssen damit auch nicht verschiedene Programme und Formate nutzen, wie das sonst nötig wäre: .pdf für Aufgaben, .java für Lösungen und ggf. .txt oder .doc für eigene Notizen.

Das erspart Ihnen 5Code: Hier ist immer zusammen, was zusammengehört.

Wenn Sie später, im Laufe des ersten Semesters oder auch in den folgenden Semestern Ihre Lösungen noch einmal nachsehen möchten, oder wenn Sie sich auf die Prüfung vorbereiten, werden Sie sich freuen, dass Sie nicht in mehreren Dokumenten alles zusammensuchen müssen. 

Code direkt ausprobieren

Nun geht es konkret um das Codieren in 5Code d.h. um das Schreiben und Ausführen von Java-Code.

Code mit vielen bisher unbekannten Anteilen

Das typische erste Beispiel-Programm gibt "Hello World" auf die Konsole aus. Dafür schreiben Sie das Programm in einem Editor, compilieren es mit dem Programm javac und lassen es mit mit dem Programm java laufen. 

Der Code dieses Programms sieht so aus:

class HelloWorld {
   public static void main(String[] args) {
      System.out.println("Hello World");
   }
}

Alles, was das Programm tut, ist "Hello World" auf die Konsole zu schreiben. Das geschieht in der Zeile

      System.out.println("Hello World");

Damit das funktioniert, müssen Sie als darum herum einen "Code-Rahmen" schreiben, der etliche "Zauberworte" beinhaltet, die Anfängern zunächst überhaupt nichts sagen:

class HelloWorld {
   public static void main(String[] args) {
        // hier steht dann der Code, der "wirklich etwas tut"
   }
}

Bis zum Ende des Semesters haben Sie natürlich alle notwendigen Konzepte kennengelernt, und können jedes einzelne Wort bzw. Zeichen erklären. 

Bis dahin ist das Schreiben dieser "Zauberworte" allerdings ein Handicap für Anfänger:

  • Sie schreiben etwas, das Sie (noch) nicht verstehen. 
    •     Das Ziel des Kurses OOP1 ist aber natürlich, dass Sie immer verstehen, was Sie tun
  • Sie werden von der eigentlichen Aufgabe abgelenkt.
  • Sie können dabei Fehler machen, die Sie nicht verstehen und daher auch nicht korrigieren können

Um sich zu Beginn auf die eigentliche Aufgabe konzentrieren zu können, nämlich die Zeilen zur Lösung der Aufgabe, wäre es sehr hilfreich, wenn Sie den Code-Rahmen auch weglassen könnten. Genau das bietet 5Code: 

  • Sie schreiben nur noch die wesentlichen Zeilen
  • 5Code ergänzt automatisch den Rahmen

Code ohne Rahmen-Code schreiben - direkt im (Aufgaben-)Text

Um ausführbaren Code mit diesen angenehmen Eigenschaften zu schreiben, reicht eine einfache Text-Formatierung nicht, die wir oben schon gesehen haben. Wir brauchen ein spezielles Element, den "Code-Block", der neben der Schriftart viele zusätzliche Funktionen bietet, die beim Codieren unterstützen. Diese Funktionen und Eigenschaften lernen Sie mit der Zeit noch kennen. 

Hier steht, direkt im normalen Text, ein solcher Code-Block:

  System.out.println("Hello World!");

In diesem Code-Block steht nur die Zeile, die etwas tut: 

    System.out.println("Hello World");

ohne den zusätzlichen Code-Rahmen.

Damit bietet Ihnen 5Code zwei Vorteile 

  • Sie schreiben nur den im Moment wesentlichen Code
  • Sie schreiben den Code direkt in den normalen Text

Code im Code-Block ausführen

Der Code soll aber nicht nur aufgeschrieben, sondern auch ausgeführt werden.

Übrigens: Alle Texte, die wie im folgenden Beispiel markiert sind, zeigen, dass Sie nun etwas tun sollen, nicht nur lesen und verstehen.

Sie sollten nun den Code im folgenden Code-Block ausführen

// Test
System.out.println("Hello World huhu");

Das tun Sie, indem Sie

  • mit dem Cursor in den Code oben gehen (mit den Pfeiltasten laufen oder mit der Maus hineinklicken).
  • Dann erscheinen oben am Code-Block ein paar grüne Icons: 

  Mit diesen Icons können Sie den Code in dem Code-Block ausführen.

  • Klicken Sie nun auf das Run-Icon, das - aus gutem Grund - aussieht, wie ein Play-Button: 

Dann wird der Code im Codeblock mit den oben beschriebenen "Zauberworten" ergänzt, compiliert und ausgeführt.

Was tun bei Fehlermeldungen, die nicht auf Ihren eigenen Code zurückzuführen sind

Wo erscheint die Ausgabe?

Nun passiert etwas an einer anderen Stelle: Jede Ausgabe taucht in der sogenannten Konsole auf.

Die Konsole finden Sie am unteren Rand des Fensters. Mit der inzwischen dort dargestellten Ausgabe sieht die Konsole nun so aus:

  • Die grünen Zeilen sind Meldungen von 5Code, dass es etwas tut. In diesem Fall:
    • Der Code wird kompiliert ("Kompiliere _Einführung__Codieren.java")
    • Erfolgsmeldung ("Kompilierung abgeschlossen")
    • Ausführung des Codes ("Starte _Einführung Einführung__Codieren")
    • Erfolgreich beendet ("Anwendung erfolgreich ausgeführt")
  •  In Schwarz ist die Ausgabe unseres kleinen Programms dargestellt ("Hello World")

Code ändern und testen

Nun können Sie selber den Code oben ändern und dann laufen lassen. 

  • Klicken Sie in den Code-Block oben
  • Editieren Sie den Code, z.B.:
    • Fügen Sie eine weiter Zeile mit einer Ausgabe-Anweisung hinzu
    • Ändern Sie den ausgegebenen Text
  • Klicken Sie auf den "Play"-Button
  • Sehen Sie nach, was unten in der Konsole angezeigt wird
Wie man einen Code-Block erstellt, wird im nächsten Kapitel erläutert.

Mit Fehlern umgehen

Beim Schreiben und Ändern von Code macht man immer wieder Fehler. 

Das kann man nicht verhindern, man kann aber lernen, mit ihnen umzugehen und sie zu beheben. 

Im folgenden Code-Block steht fehlerhafter Code, wie er typischerweise durch Verschreiben ("printn" statt "println") zustande kommt:

System.out.printn("Hello World");

 

Anzeige von Fehlern

Wenn Sie den o.a. fehlerhaften Code-Block ausführen (mit seinem Play-Button), steht in der Konsole nicht "Hello World", wie oben, sondern roter Text mit Fehlermeldungen. In diesem Fall sieht die Ausgabe auf die Konsole z.B. so aus:

Wenn mehr als eine Fehlermeldung in die Konsole geschrieben wird, können Sie zurückscrollen, mit der Scrollbar am rechten Rand, oder auch mit dem Mausrad - nachdem Sie in die Konsole geklickt haben.

Oder aber:

Wenn Sie die Konsole vergrößern wollen, können Sie sie an ihrem oberen (dicken) Rand anfassen und nach oben und unten ziehen.

Fehlerbeschreibung verstehen

Die Beschreibung des Fehlers stammt vom Compiler und ist so ausführlich, wie es dem Compiler möglich ist. 

Zu Beginn werden Sie ohne Vorkenntnisse nicht alles verstehen, wie der Compiler den Fehler beschreibt. Sie sollten sich aber darum bemühen. 

Häufig steht dort z.B.

  • welcher Begriff ("Symbol") dem Compiler nicht bekannt ist (cannot find Symbol)
    •     oft bedeutet das, dass Sie sich schlicht verschrieben haben, wie im Beispiel oben
  • dass eine offene oder geschlossene Klammer fehlt
    •     die Rolle der verschiedenen Klammern lernen Sie später
  • dass ein Semikolon erwartet wurde (';' expected)
    •     Spoiler: Ein Semikolon ";" schließt eine Anweisung ab.
  • und darunter die Stellle in der Zeile, an der der Fehler gefunden wurde
    •                         ^      <- das ist der Pfeil auf die fehlerhafte Stelle

Fehlerbeschreibung in der Code-Zeile lesen

Sie müssen nicht immer zwischen Code und Konsole hin und her schauen, die Fehlerbeschreibung ist auch direkt am Tatort zu finden: Fahren Sie mit dem Mauscursor über die rote Fehlermarkierung. Dann erscheint als "tooltip" die komplette Fehlerbeschreibung, wie sie auch in der Konsole steht:

Die Fehlermeldung im Beispiel besagt, dass das symbol printn nicht gefunden wurde. Der Grund dafür ist der, dass statt println versehentlich printn geschrieben wurde. Der Pfeil ^ zeigt direkt auf die Stelle, an der der Fehler aufgetreten ist. 

Fehler korrigieren

Nun, da Sie die Stelle mit dem Fehler gefunden haben, korrigieren Sie sie direkt im Code. Als Service hier nochmal der gleiche Code, damit Sie nicht so viel scrollen müssen: 

// Korrigieren Sie hier den fehlerhaften Code:
System.out.printn("Hello World");

Dann führen Sie den Code noch einmal aus. 

Entweder haben Sie den Code korrigiert, und es wird in der Konsole "Hello World" ausgegeben, oder Sie haben einen neuen Fehler eingebaut - was nicht selten passiert. Korrigieren Sie so lange, bis Ihr Code läuft. 

  • Computer sind schnell aber dumm:
    • Beim kleinsten Fehler im Code
    • tun sie nicht, was Sie gemeint haben,
    • sondern immer genau das, was Sie geschrieben haben.

Springen zur fehlerhaften Zeile

Es ist günstig, wenn Sie direkt von der Fehlermeldung in der Konsole zum Ort des Fehlers in Ihrem Code springen könnten. Sie brauchen dann nicht lange zu raten, wo der Fehler im Code aufgetreten ist (In unseren Beispiel gibt es nur eine Zeile, im Verlaufe des Praktikums kommen aber einige Dutzende oder sogar Hunderte Zeilen zusammen.).

Das Feature "Error-Link in der Konsolenausgabe" bietet auf der linken Seite der Konsole Informationen, wo der Fehler aufgetreten ist. Dieses

Dabei bedeuten

  • _3_Einfue... den Namen des Dokuments
  • 1 ist die Zeile, in der der Fehler aufgetreten ist.

Dass die 1 unterstrichen ist, weist darauf hin, dass sie ein Link ist. Wenn Sie auf die 1 klicken springt der Cursor im Text in den richtigen Code-Block an die Zeile mit dem Fehler:

Sie sollten dort nun zwei Markierungen finden:

  • Die Zeile, in der der Cursor steht, ist hellblau hinterlegt
    • Das ist immer so, nicht nur im Fehlerfall. So finden Sie immer schnell die Cursorzeile.
  • In der Zeile, in der der Fehler aufgetreten ist, finden Sie am rechten Rand eine rote Markierung
    • Wenn es Fehler in mehreren Zeilen gibt, gibt es in jeder Zeile eine Markierung
