Skills und Skill-Tree

Skills

Im Praktikum sollen Sie durch die Bearbeitung und Lösung von Aufgaben Kompetenzen ("Skills") in Objekt-Orientierter Programmierung erwerben.

Skill-Attribute

Außerdem können Skills mit Attributen versehen werden:

  • T = Testatrelevant. Diesen Skill müssen Sie beherrschen, um das Testat für diese Aufgabe zu bekommen.
  • W = Wiederholung. Dieser Skill wird in der Aufgabe wiederholt.
  • N = Neu. Dieser Skill wird in dieser Aufgabe das erste Mal erworben.

Ein Skill kann zum Abhaken vorbereitet sein, dann bietet er eine Checkbox an. Oder er wird nur zur Information aufgeführt werden.

Außerdem geben die Farben an, worauf der Schwerpunkt der Bewertung liegt:

  • Lila: Das Konzept, z.B. der for-Schleife soll verstanden sein
  • Purpur: Die Syntax, die Anwendung in der Programmiersprache soll korrekt sein

Eigenverantwortlich bewerten

Wenn Sie meinen, dass Sie einen Skill beherrschen, z.B. weil Sie die Aufgabe richtig gelöst haben, dann können Sie ihn abhaken. Das tun Sie in Eigenverantwortung, cheating ist also nicht nötig. 

So bekommen Sie selber einen Überblick über das, was Sie bereits beherrschen und woran Sie noch arbeiten sollten. 

Skill-Tree

Wie Sie das vielleicht von dem einen oder anderen Computerspiel kennen, werden die Skills in einem Skill-Tree dargestellt. Damit sehen Sie alle Skills über alle Aufgaben hinweg im Überblick. 

Den Skill-Tree erhalten Sie auf der Startseite nach dem Klick auf den Button ganz unten rechts.

Die Darstellung des Skill-Trees ist noch in Überarbeitung und wird sich in den kommenden Versionen von 5Code  noch ändern und verbessern.

Zusammenfassung

Sie haben nun einige wichtige Grundbegriffe und Konzepte kennengelernt, sowohl zur Programmierung allgemein, als auch zur Nutzung von 5Code:

  • Bearbeiten von Aufgaben als Projekte im Workspace von 5Code
  • Setzen des Pfads zum Workspace und zum Java-Compiler
  • Schreiben von formatiertem Text und einfachen Grafiken
  • Schreiben und Ausführen von Code im Text-Dokument
  • Erstellen und Ausführen von Code 
  • Nutzung der Konsole
  • Nutzen von Features und 5Coins
  • LVÜAC - Lesen-Verstehen-Überlegen-Verstehen-Aufschreiben-Codieren
  • Vermeiden von Code-Duplikation
  • Nutzen und Anzeigen von Skills

Darauf können wir dann in den nächsten Terminen aufbauen.
