Projekte im Workspace organisieren

Hier bekommen Sie eine kurze Einführung in die Arbeit mit Projekten im Workspace von 5Code

  • Arbeiten mit Aufgaben = Projekten
  • Setzen des Workspace-Ordners
  • Speichern von Projekten

Projekt

Alle Dokumente, die zu einer Praktikums-Aufgabe zusammengehören, d.h. 

  • Aufgaben
  • Erläuterungen
  • Lösungen
  • etc

werden zu einem "Projekt" zusammengefasst.

Dieses Konzept finden Sie auch in den meisten anderen Integrierten Entwicklungsumgebungen (IDEs).

Workspace

Der "Workspace" ist der Standard-Ordner, in dem alle Ihre Projekte gespeichert werden. 

Es empfiehlt sich, für das Praktikum OOP1 einen eigenen Ordner anzulegen - wo, hängt von Ihrer eigenen File-Organisation ab. 

Wenn Sie 5Code das erste Mal öffnen, müssen Sie einen lokalen Ordner in Ihrem Dateisystem angeben (oder neu anlegen), in dem der Workspace untergebraucht wird. 

(Wenn Sie aus Gründen den Workspace-Ordner später ändern möchten, können Sie das auf der Startseite tun. Ihre Projekte bleiben dann aber im alten Workspace-Ordner.)

Projekte aus der Cloud

Die Projekte mit allen Aufgaben werden Ihnen über eine Cloud, d.h. über einen File-Server, zur Verfügung gestellt.

Mit der Cloud haben Sie sich bereits beim ersten Start von 5Code verbunden. Dabei wurde Ihnen genau erklärt, was Sie dafür machen müssen. Diese Verknüpfung müssen Sie auch nur einmal zu Beginn des Semesters machen.  

Auf der Startseite von 5Code werden Ihnen angezeigt

  • Alle Projekte, die Sie bereits auf Ihren Rechner heruntergeladen haben
  • Projekte, die in der Cloud zum Download bereitstehen, was durch dieses Icon symbolisiert wird: 
    • Diese können Sie durch einen Klick auf den entsprechenden Button herunterladen
    • Dann liegen diese Projekte auf Ihrem Rechner in dem lokalen Ordner

So kommen Sie zur Startseite (klicken Sie auf das (i) oder diesen Text um mehr zu erfahren).

Ihre Projekte und Dateien speichern Sie immer NUR lokal auf Ihrem  Rechner. 

Die Cloud-Anbindung dient NUR dazu, dass Sie Projekte herunterladen können. Nichts von Ihren Daten wird in die Cloud hochgeladen, weder automatisch, noch von Ihnen.

Wie Sie Projekte zwischen Rechnern (zu Hause, PC-Pool, ...) transferieren, wird weiter unten erklärt.

Aufgaben und Texte

Wenn Sie ein Projekt auf der Startseite anklicken, wird das Projekt geöffnet.

Dann erscheinen in der Navigation links die Dokumente in diesem Projekt:

Welches Dokument in welchem Projekt zuerst geöffnet werden sollte, sollte aus der Benennung der Dokumente klar werden. 

Lösungen

Ihre eigenen Lösungen schreiben Sie zunächst am einfachsten direkt in die Aufgaben-Dokumente hinein. 

  • sowohl Ihre eigenen Anmerkungen und Überlegungen
  • als auch Ihren Java-Code

Wie das geht, erfahren Sie in den folgenden Dokumenten.

Speichern

Sie speichern immer alle Dokumente des Projekts zusammen ab, um die Konsistenz aller Dokumente zu gewährleisten. Das geht über

  • Das Menü Projekt -> Projekt speichern
  • oder den Shortcut (das Tastaturkürzel) Strg-S (bzw. Cmd-S auf MacOS)

Dass Sie ein Dokument geändert haben, sehen Sie in der Titelleiste: Dort wird dann ein Punkt neben dem Namen angezeigt:

Ihre Projekte und Dateien speichern Sie immer NUR lokal auf Ihrem Rechner. 

Speichern unter

Wenn Sie länger an einem komplizierteren Projekt, d.h. einer Praktikums-Aufgabe, arbeiten, dann lohnt es sich ggf., den Stand zu speichern und eine Kopie des Projekts weiter zu bearbeiten. Die Kopie können Sie dann z.B. mit einer fortlaufenden Versionsnummer ergänzen, z.B. "Aufgabe 1 v12".

Eine Kopie des Projekts erhalten Sie über 

  • Das Menü Projekt -> Projekt speichern unter
  • oder den Shortcut (das Tastaturkürzel) Strg-Shift-S (bzw. Cmd-Shift-S auf MacOS)

So können Sie immer auf den Original-Stand das Projekts zurückgreifen.

Transfer von zu Hause in den PC-Pool auf den Campus - und zurück

Sie bearbeiten die Aufgaben zu Hause an Ihrem eigenen Rechner, bereiten Ihre Lösungen vor und speichern Sie auch auf Ihrem eigenen Rechner. 

Im Präsenz-Termin der Praktika im PC-Pool auf dem Campus brauchen Sie dann natürlich Ihre Lösungen. Um diese hin- und auch wieder zurück zu transferieren, gibt es zwei Möglichkeiten, die im Folgenden erläutert werden.

Mit dem eigenen Laptop

Wenn Sie Ihren Laptop dann mit in den PC-Pool auf den Campus mitnehmen, können Sie den dort auch weiter benutzen (natürlich unter Beachtung der dortigen Randbedingungen). Dann haben Sie immer alle Ihre Daten und Projekte dabei und können sie überall bearbeiten. 

Mit einem USB-Stick 

Wenn Sie Ihren Rechner nicht von zu Hause in den PC-Pool mitbringen können (Desktop-Rechner) oder wollen, können Sie Ihre Projekte auch per USB-Stick hin und zurück transferieren.

  • Kopieren Sie dazu einfach den betreffenden Projekt-Ordner auf den USB-Stick 
  • Transferieren Sie immer komplette Projekt-Ordner!
    • Andernfalls kann es zu Konsistenz-Problemen kommen

Auf den PC-Pool-Computern richten Sie dann beim ersten Mal 5Code noch einmal ein: 

  • Workspace (falls nicht schon automatisch eingerichtet)
  • Java (falls nicht schon automatisch eingerichtet)
  • Benutzernamen !
    • geben Sie auf jeden Fall den gleichen Benutzernamen an, wie zu Hause. Andernfalls können Sie nicht Ihre Erfolge, 5Coins, Features, Skills, etc. übertragen. Was das ist, wird in den folgenden Kapiteln erläutert.

Fast immer bearbeiten, korrigieren und ergänzen Sie Ihre Projekte auch während des Präsenz-Termin s

Denken Sie auch daran, Ihr im PC-Pool bearbeitetes Projekt wieder zurück vom PC-Pool mit nach Hause zu nehmen, damit Sie zu Hause immer die aktuelle Version mit allen Ihren Lösungen, Features, etc. haben. Auch hier transferieren Sie immer das komplette Projekt!

Tipp: Speichern Sie Ihre Überarbeitungen im PC-Pool unter einem anderen Namen.
