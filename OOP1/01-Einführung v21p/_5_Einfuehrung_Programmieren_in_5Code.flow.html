Einführung in das Programmieren

Programme schrittweise erstellen 

Wie bereits besprochen, versteht man unter Programmieren man den kompletten Weg vom Problem zum Programm.

Es kommt praktisch nie vor, dass man seinen Code spontan und sofort nach dem Lesen der Anforderungen erstellt. Praktisch immer macht man das in mehreren Schritten - weil man schrittweise Anforderungen liest, versteht, Lösungen überlegt, notiert und dann codiert. 

Damit ist auch schon fast das ganze, sehr komplexe Gebiet des Software-Engineerings beschrieben. 

Die dort definierten Phasen Analyse, Design, Implementierung lassen sich leichter verständlich auch so benennen:

Lesen -> Verstehen -> Überlegen -> Aufschreiben -> Codieren

oder auf Englisch:

Read it -> Get it -> Think it -> Note it -> Code it

Diese Vorgehensweise wird im Folgenden mit LVÜAC abgekürzt. Sie werden ihr in OOP1 und OOP2 und hoffentlich ihr ganzes Berufsleben lang folgen ;-).

LVÜAC in 5Code

Da LVÜAC die universelle Vorgehensweise beim Programmieren ist, ist es günstig, dass sie von 5Code besonders gut unterstützt wird:

  • Lesen und Verstehen - Aufgaben liegen in editierbarer Form vor
    • Sie können z.B. mit den Textmarker-Funktionen wichtige Teile markieren
    • Das geht einfach mit den Icons am oberen Fensterrand:  
    • Die Tooltips der Icons geben auch an, mit welchen Tastatur-Kürzeln (shortcuts) formatiert werden kann. 
  • Überlegen und Aufschreiben
    • Sie können direkt in die Aufgaben Ihre Überlegungen und Notizen hineinschreiben
    • Damit man Ihre Notizen auch leicht wiederfindet, markieren Sie sie ebenfalls.
  • Codieren
    • Code wird direkt im Anschluss an Ihre Notizen und die konkrete Aufgabenstellung geschrieben

So ist der Zusammenhang immer gegeben:

  • Was soll der Code tun - die Aufgabenstellung
  • Wie tut Ihr Code tatsächlich - Ihre eigene Lösung 

Dieser Zusammenhang ist immer wichtig, wenn der Code nicht funktioniert. 

  • Dann können Sie immer die Anforderung (SOLL) mit Ihrer Umsetzung in Code (IST) vergleichen

Eine schlechte Lösung wäre, einen weiteren Code-Block zu machen, und die Anweisungen von oben dort hinein zu kopieren. 

Kopieren ist meistens schlecht, das gibt "Code-Duplikation" - was sehr fehleranfällig ist:

  • Man kopiert nicht genug oder zuviel
  • Nach dem Kopieren ergänzt man an einer Stelle, vergisst aber die Kopie
  • Man modifiziert die Kopie und denkt fälschlicherweise, auch das Original modifizieren zu müssen

Es gibt viele Lösungen, ohne Code-Duplikation auszukommen. Viele davon lernen wir in OOP1 und auch noch in OOP2 kennen.

Aber auch jetzt schon bietet 5Code eine Lösung:

    -> Wir lassen einfach beide Code-Blöcke hintereinander ausführen.

Dazu klicken Sie in den zweiten Code-Block, den mit den Anweisungen für das Menü, um die Ausführen-Icons erscheinen zu lassen. Dann haben Sie zwei Möglichkeiten:

  • Sie klicken auf das zweite Execute-Icon 
  • Sie nutzen das Tastaturkürzel Strg-Shift-E

Probieren Sie beide Möglichkeiten aus, dann werden alle Code-Blocks bis zu dem aktuellen Code-Block (einschließlich) ausgeführt.

Auf der Konsole erscheint nun die Ausgabe aus beiden Code-Blocks:

Mehrere Code-Blöcke ausführen

Auf geschilderte Art und Weise können Sie also beliebig viele Code-Blöcke bis zum aktuellen Code-Block (einschließlich) ausführen:

  • Sie klicken auf das zweite Execute-Icon 
  • Sie nutzen das Tastaturkürzel Strg-Shift-E

Das ist für die schrittweise Entwicklung von Code sehr praktisch - schon jetzt am Anfang, und auch später, wenn Sie umfangreichere Programme schreiben. 

Schließlich gibt es noch die dritte Möglichkeit, alle Code-Blöcke des aktuellen Dokuments auszuführen.

  • Sie klicken auf das dritte Execute-Icon 
  • Sie nutzen das Tastaturkürzel Alt-E

Das ist praktisch, wenn Sie alle Aufgaben in einem Dokument fertiggestellt haben und das komplette Ergebnis ausführen lassen wollen. 

Nutzung der Konsole

Alle Text-Ausgaben erscheinen immer auf der Konsole. Was erscheint, schreiben Sie in Ihr Programm, wie wir es oben getan haben. Die Ausgaben können Sie in der Konsole nicht editieren.

Sie können den Text in der Konsole, wenn Sie möchten, kopieren und in einem anderen Text wieder einsetzen. Das ist ganz günstig, um seine Ergebnisse zu dokumentieren:
