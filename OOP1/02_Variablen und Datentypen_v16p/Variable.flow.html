Kommentare schreiben

Um den Code zu beschreiben, werden Kommentare verwendet.

Das ist immer wichtig und sinnvoll, da Sie immer wissen sollten, was Ihr Code tun soll.

1) Sie schreiben auf, was der Code tun SOLL - in einem Kommentar

2) Sie schreiben den Code, das IST das Programm

3) Wenn der Code nicht tut, was er soll, können Sie im Kommentar
    immer nachlesen, was er eigentlich tun sollte.
    D.h. Sie können das IST gegen das SOLL prüfen

Das Schreiben von Kommentaren entspricht den Schritten  Ü&A in der allgemeinen Vorgehensweise, die Sie in der ersten Aufgabe kennengelernt haben.:

Lesen -> Verstehen -> Überlegen -> Aufschreiben -> Codieren

post/pre - increment - Operator

Nun kommen wir zu spezielleren Operatoren: Um das Erhöhen eines Wertes um 1 abzukürzen (für faule Programmierer :-) gibt es einen speziellen Inkrement-Operator:

Statt pi = pi + 1; kann man auch  pi += 1; schreiben oder auch ganz kurz: pi++. Dabei ist ++ der Inkrement-Operator.

int pi;     // Definition einiger Zahl-Variablen
int pj;

pi = 1;
pi++;            // Variable pi incrementieren (d.h. um 1 erhöhen)
            //  !! geben Sie nun den Wert von pi aus:

System.out.println(pi);      // -> Ausgabe:
      //

Ergänzen Sie im Codeblock oben Anweisungen, um den Wert der Variablen pi zuerst zu inkrementieren und geben Sie ihn dann auf die Konsole und führen Sie den Codeblock dann aus. Dokumentieren Sie in ihren Code das Ergebnis.

Sie können die Ausgabe in der Konsole auch kopieren und in den Code (oder Text) einfügen.

Haben Sie das Ergebnis verstanden?

Antwort: Ja.

pi++ ist zusätzlich ein "Ausdruck", der immer dort eingesetzt werden kann, wo auch pi erlaubt ist. z.B. so: 

int pi=42;     // Definition einiger Zahl-Variablen
int pj;

pj = pi++;  //  !! geben Sie nun pi und pj aus:


System.out.println(pi);      // -> Ausgabe:
      //

Ergänzen Sie im Codeblock oben eine Anweisung, um den Wert der Variablen pi auf die Konsole auszugeben und führen Sie den Codeblock dann aus. Dokumentieren Sie in ihren Code das Ergebnis.

Haben Sie das Ergebnis verstanden?

Antwort: Ja, weil von rechts nach links gearbeitet wird. Erst wird pi inkrementiert und danach pj zugewiesen.

Da der Operator ++ nach dem Variablennamen pi geschrieben wird, deutet das darauf hin, dass der Wert von pi erst nach der Verwendung des ganzen Ausdrucks pi++ incrementiert wird. pi++ heißt daher auch "post-increment-operator". ("post" = lateinisch für "danach")

Dementsprechend gibt es auch einen pre-increment-operator: ++pi.

int pi=42;     // Definition einiger Zahl-Variablen
int pj;

pj = ++pi;  //  !! geben Sie nun pi und pj aus

System.out.println(pi);      // -> Ausgabe:
      //

Ergänzen Sie im Codeblock oben eine Anweisung, um den Wert der Variablen pi auf die Konsole auszugeben und führen Sie den Codeblock dann aus. Dokumentieren Sie in ihren Code das Ergebnis. 

Haben Sie das Ergebnis verstanden?

Modulo

Ein eher selten verwendeter, aber nützlicher, Operator ist der Modulo-Operator %, der den Rest einer ganzzahligen Division angibt. Beispiel: 8 % 3 ist 2

Ergänzen Sie den unten stehenden Code, zu einem kleinen Programm, das eine Fließkommazahl auf den nächsten Hunderter abrundet.

Beispiele: Ein Originalwert von 123.34 ergibt abgerundet 100.00, ein originalwert von 4567.39 ergibt abgerundet 4500.00

/** * Der folgende Code zeigt beispielhaft,  
* wie Sie mit LVÜAC zum Ziel kommen
* Die Überlegungen sind bereits als Kommentar geschrieben
* Sie ergänzen nun zwischen die Kommentarzeilen  
* Ihren Code für die ausführbare Lösung. */

// Definieren einer Variable für den Originalwert
// als Fliesskommazahl
double originalwert = 123.34;      

// Eine Variable für den abgerundeten Wert
double abgerundet;

// Eine Variable für die differenz zwischen orginalwert und abgerundetem Wert
double differenz;

// Die differenz wird nun berechnet als Rest einer Modulo-Berechnung
differenz = originalwert % 100;

// Jetzt berechnen wir den abgerundeten Wert mit Hilfe der differenz
abgerundet = originalwert - differenz;

// Zum Schluss geben wir den abgerundeten Wert auf die Konsole aus    
System.out.println("abgerundet: " + abgerundet);

Sie brauchen die Lösungen aber nicht für das Testat vorweisen.

Polynomberechnung 

Ein weiteres Beispiel für Rechenoperationen findet sich in der Berechnung von CRCs (Cyclic Redundany Checks). Diese werden z.B. in Protokollen für Netzwerk-Transfers oder für die Daten auf CDs berechnet, um feststellen zu können, ob Fehler in Daten vorgekommen sind. Für die Berechnung werden sogenannte „Generatorpolynome“ bestimmt.

Schreiben Sie ein Programm, das den Wert eines Polynoms als Fließkommazahl berechnet 

 (x3 bedeutet dabei die Potenzierung: x hoch 3).

               genpoly = a*x12 + b*x11 + c*x3 +d*x2 + e

1)     Überlegen Sie zuerst, wie die Lösung aussieht. 

2)     Beschreiben Sie dann Ihr Vorgehen im Code mit //-Kommentaren, 

3)     bevor Sie danach den ausführenden Java-Code dazu schreiben !!

ganz so, wie Sie es in der vorangegangenen Aufgabe getan haben.

Definieren Sie für den Parameter x, die Koeffizienten a-e und das Ergebnis y jeweils eine Variable mit einem geeigneten Datentyp. 

Setzen Sie die Variablen auf verschiedene Werte, berechnen Sie das Polynom an dieser Stelle und geben Sie das Ergebnis auf die Konsole aus.

- Verwenden Sie die vier Grundrechenarten mit den Operationen + - * und /

- Verwenden Sie Klammern um die Reihenfolge der Berechnung eindeutig und lesbar zu machen

Hinweis: der Operator ^ existiert zwar für Zahlen, er berechnet allerdings keine Potenzen. Wir hatten ihn an einer anderen Stelle kennengelernt.

Optimierung

Versuchen Sie, nachdem Sie eine einfache Lösung der Aufgabe hinbekommen haben, die Lösung auf geschicktere Art und Weise zu berechnen.  D.h. versuchen Sie, mit so wenig Multiplikationen wie möglich auszukommen. 

Lösen Sie diese Aufgaben in einem neuen Codeblock. Schreiben Sie dabei immer als Kommentar auf, WAS sie im nächsten Schritt codieren wollen, dann schreiben Sie als Code hin, WIE sie es tun.
