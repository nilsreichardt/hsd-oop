Als Beispiel für Referenz-Datentypen verwenden wir den Datentyp String. Unter Zeichenketten kann sich jeder etwas vorstellen, und Sie können Strings auch als Literal angeben, was den Umgang damit sehr vereinfacht. Später lernen wir noch eine Menge anderer Referenz-Datentypen kennen.

Skizzieren Sie für jede folgende Unteraufgabe auf einfache Art und Weise, was beim Definieren, Zuweisen und Arbeiten mit den Strings jeweils im Speicher steht – wie wir es in der Vorlesung gesehen haben. Dafür ergänzen Sie die gegebenen Zeichnungen hier im Dokument.

Wenn die folgende Java-Codezeile ausgeführt wurde

String name = "Heinz";

ergeben sich folgende Werte im Speicher:

  • Dabei ist name der Namen der Variable
  • Der Wert der Variablen ist ein Verweis auf eine Speicherstelle, ab der ein String im Speicher liegt.

Technisch ist der Verweis natürlich kein gezeichneter Pfeil sondern eine Speicheradresse, wie in der Vorlesung behandelt. Hier ist die Darstellung mit Pfeilen aber ausreichend und übersichtlicher. 

Definieren von zwei Variablen, die als Datentyp einen Referenzdatentyp haben.

Definieren Sie in einem neuen Code-Block zwei String-Variablen name und vorname. Weisen Sie ihnen aber noch keinen (!) Wert zu.

String vorname;
String nachname;

 

Frage: Was steht nun als Wert im Speicher?

Hinweis: Es kann nie „nichts“ im Speicher stehen.

Skizzieren Sie die entsprechenden Werte im Speicher in der folgenden Zeichnung 

Es sind zwar die beiden Variablen im Speicher vermerkt, jedoch haben diese noch keine Referenz, weil diesen kein Wert zugewiesen wurde.

Schisserprinzip – Initialisierung wird erzwungen

Geben Sie beide Variablen auf die Konsole aus.

Programmieren Sie das in einem weiteren Codeblock:

String vorname;
String nachname;

System.out.println("vorname: " + vorname);
System.out.println("nachname: " + nachname);

// Fehler: "variable nachname might not have been initialized"

 

Dann versuchen Sie, diesen und alle vorherigen Code-Blöcke (in denen die Definitionen der beiden Variablen stehen) auszuführen.

Der Compiler lässt Sie diesen Code nicht compilieren, da er dem Schisserprinzip folgt: Sie haben die Variablen nicht initialisiert (d.h. nicht auf einen Anfangswert gesetzt), also ist nicht klar, was der Wert dieser Variablen sein soll.

null – definierter Wert für „es wurde noch kein Wert gesetzt“

Initialisieren Sie nun die Variable name auf den Wert „definiert auf keinen Wert gesetzt“ (null), den wir in der Vorlesung kennen gelernt haben. Geben Sie nun name auf die Konsole aus.

Die Fehler im Codeblock in 1.2 sollen hier die Compilierung nicht behindern
Kommentieren Sie den fehlerhaften Code aus.

Dazu setzen Sie z.B. vor jede Zeile die Zeichen //.

Sie können auch mit einem Klick den kompletten Code-Block auskommentieren.

Um den kompletten Code-Block "auszukommentieren", gibt es in der Toolbar des Code einen Button //. Probieren Sie ihn aus. 

Mit dem gleichen Button können Sie auch dieses "auskommentieren" wieder rückgängig machen.

Tun Sie das, in einem eigenen Codeblock und führen Sie alle Code-Blöcke aus:

String vorname = null;
String nachname = null;

System.out.println("vorname: " + vorname);
System.out.println("nachname: " + nachname);

 

  • Funktioniert das nun?
    • Ja
  • Was wird ausgegeben?
    • Es wird "vorname: null" und "nachname: null" ausgegeben
  • Inwiefern ist für den Compiler die Situation nun unterschiedlich zu oben in 1.2.?
    • Wir haben dem Compiler nun explizit mitgeteilt, dass diese Variable noch keinen konkreten Wert hat, sondern nur "null" ist. Damit weiß der Compiler, dass wir uns Gedanken um diese Variable gemacht und diese beabsichtigt ist.

Zuweisung ist nicht Kopieren (jedenfalls nicht bei Referenz-Datentypen)

Weisen Sie nun vorname der Variablen name zu (name = vorname) und geben Sie name und vorname auf die Konsole aus.

Konkatenieren von Strings

Hängen Sie nun die beiden Strings name und vorname aneinander (der Fachausdruck dafür ist „konkatenieren“ oder auf Englisch „concatenate“) und geben Sie sie aus.

Wir hatten einen Konkatenations-Operator kennengelernt

Tun Sie das, in einem eigenen Codeblock:

String vorname = "Max";
String nachname = "Mustermann";

System.out.println("Vorname: " + vorname + "; Nachname: " + nachname);

// Da hier geht auch:
String name = vorname + " " + nachname;
System.out.println(name);

 

Automatische Umwandlung in Strings

Der Konkatenierungs-Operator kann auch dazu verwendet werden, um Strings mit Ausdrücken anderer Datentypen als String zu verknüpfen. Das Ergebnis ist dabei immer ein String.

Konkatenieren Sie name und String-Literale mit Zahlen verschiedener Datentypen sowie mit char- und boolean-Werten und geben Sie das Ergebnis auf der Konsole aus.

Tun Sie das, in einem eigenen Codeblock:

String vorname = "Max";
String nachname = "Mustermann";

byte alter = 24;
char geschlecht = 'm';
boolean hatAbitur = true;
double kontostand = 231.23;
    
System.out.println("Vorname: " + vorname + "; Nachname: " + nachname + "; Alter: " + alter + "; Geschlecht: " + geschlecht + "; hatAbitur: " + hatAbitur + "; Kontostand: " + kontostand);

 

Dass das funktioniert, bedeutet, dass die Ausdrücke anderer Datentypen in diesem Fall automagically in Strings umgewandelt werden. Praktisch ist das unter anderem, um eine „gemischte“ Ausgabe zusammenzubauen,

z.B.: System.out.println("x=" + x + " ,y=" + y + " z=" + 5);
