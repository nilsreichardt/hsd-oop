Beschreiben 

Bisher haben wir alles, was wir als Code geschrieben haben, sofort auch direkt ausgeführt - dank der Unterstützung von 5Code.

Wenn wir nun Code in Prozeduren packen, müssen wir allgemein unterscheiden zwischen:

  • der Definition der Prozedur - es wird einmal beschrieben, was in der Prozedur passieren soll
    • nur dadurch, dass eine Prozedur definiert wird, wird ihr Code noch nicht ausgeführt
  • dem Aufruf der Prozedur - es kann beliebig oft bestimmt werden, dass es passieren soll
    • erst durch den expliziten Aufruf einer Prozedur wird ihr Code auch ausgeführt.

Diese müssen an unterschiedlichen Stellen im Code positioniert werden, wie in der Vorlesung bereits dargestellt wurde.

Beachten Sie nun zuerst die Richtlinien für

  • einen guten Namen
  • eine Beschreibung
    • mit javadoc
    • mit Zeilenkommentaren

die Sie im so erweiterten Coding Style im Kapitel Beschreibung von Prozeduren und Funktionen finden.

Startprozedur - public static void main(String[] args)

Auszuführenden Code haben wir bisher in Code-Blöcke geschrieben. Im Hintergrund hat 5Code diese Anweisungen in die Prozedur public static void main gesetzt. Diese Prozedur muss in jedem Java-Programm vorhanden sein, da diese als Startpunkt jeder Java-Anwendung dient.

Öffnen Sie erneut Aufgabe_05.java und finden Sie public static void main. Dies ist hier bereits für Sie vorgegeben. (Weiter unten ist noch mehr Code vorgegeben, diesen betrachten wir zunächst jedoch nicht.)

Direkt auszuführenden Code und Tests schreiben Sie von nun an direkt in die Prozedur main() hinein, und nicht mehr in Code-Blöcke. Dafür gibt es einen neuen Aufgaben-Typ, dessen Markierung mit einem neuen Symbol endet:

Klicken Sie auf die Markierung mit dem Pfeil, um automatisch Lösungscode in der Java-Datei generieren zu lassen. Das "M" gibt dabei an, dass Ihre Lösung in main() geschrieben werden soll.

Sofort ausführbaren Code schreiben

Schreiben Sie Code, der "Hello World" auf die Konsole ausgibt in main() . Wie das geht, kennen Sie bereits. Sie schreiben jetzt diesen Code (1 Zeile) nun direkt in den body der Prozedur main().

Wie Sie feststellen, wird bei Klick auf diese Marierung automatisch in main() ein Kommentar eingefügt, nach dem Sie dann Ihren eigenen Code für die Lösung schreiben. Der Kommentar beschreibt die Stelle in der Aufgabe, die zu dem folgenden Code gehört. So haben Sie immer direkt einen Bezug zwischen Ihrer Lösung und der Aufgabe.

Definition von Prozeduren und Funktionen

Nachdem wir nun erfahren haben, wie man in Klassen ausführbaren Code schreibt und laufen lässt, kommen wir nun zur Definition von Prozeduren und Funktionen.

Prozeduren und Funktionen werden innerhalb einer Klasse definiert, zwischen dem ersten { und dem letzten } (d.h. im Body der Klasse). Ein Beispiel ist die bereits vorhandene Prozedur public static void main.

Dabei ist wichtig, dass Definitionen von Prozeduren nicht ineinander verschachtelt werden dürfen. Sie können also z.B. keine Prozeduren innerhalb von main() definieren. Alle Definitionen von Prozeduren stehen also parallel zur Definition von main() in der Klasse.

Prozedur static void begruessung()

Definieren Sie nun eine eigene Prozedur static void begruessung(), welche einen Begrüßungstext auf die Konsole ausgibt.

Klicken Sie auf die Markierung mit dem Pfeil, um automatisch an der richtigen Stelle in der Java-Datei einen Kommentar für Ihre Definition generieren zu lassen. Danach schreiben Sie dann die Definition Ihrer Prozedur. Das "D" an der Markierung gibt dabei an, dass Ihre Lösung eine Definition einer Prozedur oder Funktion ist.
Hinweis: Vergessen Sie nicht das "Zauberwort" static bei der Definition, sonst können Sie die Prozedur/Funktion nicht direkt aufrufen.

In der automatisch generierten Lösungsvorlage wird auch ein vollständiger Javadoc-Header erstellt. Ergänzen Sie diesen mit den entsprechenden Informationen und entfernen Sie ggf. nicht benötigte Tags.

 

Fehlerüberprüfung

Um den Code auf Korrektheit zu überprüfen, klicken Sie den Kompilieren-Button ("Smiley") in der Toolbar oder nutzen Sie den Shortcut (Strg-K). Durch den Kompiliervorgang können Sie den Code auf Fehler überprüfen lassen, ohne das Programm auszuführen. Die Ausgaben finden Sie wie gewohnt unten auf der Konsole.

Zusätzlich gibt es, ohne das Kompilieren anzuklicken, bereits Hinweise zu möglichen Syntax-Fehlern: Wird der Kompilieren-Button rot dargestellt, können Sie den Maus-Cursor über das Icon bewegen und erhalten nützliche Fehlerhinweise.

Überprüfen Sie bis hierhin, ob Ihr Code korrekt ist.

 

Aufruf von Prozeduren und Funktionen

Vorhin haben wir eine Prozedur definiert. Aber um die Anweisungen darin auch nutzen zu können, müssen wir sie auch aufrufen.

Dies schreiben wir innerhalb von public static void main(). Sie erinnern sich, darin befindet sich unser auszuführender Code.

Ergänzen Sie main() um einen Aufruf der Prozedur begruessung und führen Sie die Klasse anschließend aus.

Was wird auf der Konsole ausgegeben?

Es wird "Moin Moin :)" ausgegeben.

Aktiver Kommentar ///

In der automatisch generierten Lösungsvorlage in main() entdecken Sie eine neue Art Kommentar, den Aktiven Kommentar.

Der aktive Kommentar, geschrieben: ///

  • ist syntaktisch ein normaler Java-Zeilenkommentar, den Sie nutzen, um zu beschreiben, was die folgenden Codezeilen tun.
  • wird zusätzlich auf der Konsole dargestellt.

Damit haben Sie die Möglichkeit, schnell und einfach Konsolenausgaben zu codieren. Zeilen-Kommentare, die mit /// beginnen, werden intern in 5Code durch einen Aufruf von System.out.println() ersetzt.

Sie haben damit bei vielen Ausgaben auf die Konsole damit immer den Überblick, welche Ausgaben zu welchem Aufgabenteil gehören. Nutzen Sie dieses Feature, um Ihren Code und Ihre Konsolenausgaben - auf einfache Weise - verständlicher und übersichtlicher zu machen. 

Das Feature "Aktiver Kommentar" sollte sich beim Öffnen des Projekts freigeschaltet haben. Sie können es in der Feature-Liste finden und aktivieren, falls Sie dies noch nicht getan haben.

Probieren Sie den aktiven Kommentar in einem neuen Codeblock aus

// Normaler Kommentar
/// Aktiver Kommentar

.

Überladen von Prozeduren

Wir können nun eine Begrüßung auf der Konsole ausgeben. Im nächsten Schritt wollen wir diese um einen Namen erweitern. Zu diesem Zweck überladen wir nun die Prozedur begruessung.

Was bedeutet "eine Prozedur überladen"?
Gleicher Name

Man kann mehrere Prozeduren mit gleichem Namen definieren.

Eindeutig unterscheidbare Parameterlisten

Die Parameterlisten der Prozeduren sind unterscheidbar in

  • Länge und/oder
  • Datentypen der Parameter in der angegebenen Reihenfolge

Definition von überladenen Prozeduren

Definieren Sie eine Prozedur zur Begrüßung einer Person, indem Sie die Prozedur begruessung überladen. Als Parameter wird ein String für den Namen der Person genutzt.

Überladen Sie die Prozedur begruessung erneut, indem Sie noch einen int-Parameter ergänzen. Dieser bestimmt, wie oft die Begrüßung ausgegeben werden soll. Definieren Sie auch diese Prozedur zur wiederholten Begrüßung.

Zuletzt testen Sie die überladenen Prozeduren.
