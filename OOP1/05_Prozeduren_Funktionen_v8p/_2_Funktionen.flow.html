Unterschied zu Prozeduren

Funktionen sind sehr ähnlich wie Prozeduren,

  • In ihnen passiert etwas
  • Plus: Sie haben auch einen Wert als Ergebnis

Funktionen

  • Definition - es wird einmal beschrieben, was in der Prozedur passieren soll
  • Aufruf - es kann beliebig oft bestimmt werden, dass es passieren soll

Auch hier gilt: Diese müssen an unterschiedlichen Stellen im Code positioniert werden, wie Sie bereits in der erstenAufgabe gelernt haben.

Beachten Sie nun zuerst die folgenden Richtlinien für

  • einen guten Namen
  • eine Beschreibung
    • mit javadoc
    • mit Zeilenkommentaren

die Sie im erweiterten Coding Style finden.

 

Einfache Funktionen - ISBN-10

Wir entwickeln nun schrittweise mehrere Funktionen, die dann zusammen eine ISBN-10 erstellen.

Lesen und verstehen Sie zunächst die hier beschriebene allgemeine Definition einer ISBN. Implementieren Sie dann in der angegebenen Reihenfolge die Funktionen in den folgenden Aufgabenstellungen und testen Sie alle diese Funktionen schrittweise durch passend parametrisierte Aufrufe.

Beachten Sie dabei die oben angegebenen Richtlinen für gute Namen und Beschreibungen um gut lesbaren Code zu erzeugen.

Allgemein: Bücher werden mit einer internationalen Standard-Buchnummer (ISBN) versehen, damit sie international eindeutig referenziert und bestellt werden können. Die ISBN-10 besteht aus einer 9-stelligen Zahl, zu der noch eine Prüfziffer hinzugefügt wird. Mit dieser Prüfziffer kann festgestellt werden, ob die gesamte ISBN korrekt ist, oder ob eine falsche Ziffer dabei ist. (Wie die Prüfziffer genau ermittelt wird steht z.B. in https://de.wikipedia.org/wiki/Internationale_Standardbuchnummer ). In den Aufgaben unten wird der jeweils benötigte Teil angegeben.

Hinweis: Den Link können Sie per Klick im integrierten 5Code-Browser öffnen. Es ist also nicht nötig, dafür extra ein separates Programm zu öffnen!

 

Top-Down Überlegungen zum Aufbau der Berechnung

Wir brauchen also folgende Berechnungen zur Lösung der gesamten Aufgabe:

  • Berechnung einer formatierten ISBN als String
    • Berechnung einer Prüfziffer aus der Zahl
      • Ermittlung einer Ziffer an einer Stelle in der Zahl
    • Darstellung einer Prüfziffer als char

Diese Überlegung geht Top-Down vor, d.h. Wir beginnen mit der obersten (Top) Aufgabe "Berechnung einer formatierten ISBN als String" und überlegen, wie diese in untergeordneten Teil-Aufgaben umgesetzt werden kann. Dadurch arbeiten wir uns weiter nach "unten" (Down) vor - solange, bis eine Teil-Aufgaben nicht weiter zerlegt werden kann. Diese Vorgehensweise werden wir häufig verwenden, und auch noch ausführlicher darauf eingehen. 

Die Funktionen für diese Berechnungen erstellen wir in dieser Aufgabe in einer Reihenfolge, in der wir sie hier einfach testen können.

 

Ziffer an Stelle s berechnen

Wir brauchen eine Funktion, die Ziffer an der Stelle stelle einer 9-stelligen int-Zahl zahl als int-Zahl berechnet und als int-Wert zurückgibt.

Beispiel: die Ziffer an der Stelle 3 der Zahl 123456789 ist 3.

Hinweis: An die Ziffer an der Stelle 9, einer 9-stelligen int-Zahl kommen Sie mittels des modulo-Operators der den Rest einer ganzzahligen Division ermittelt. Beispiel: 123456789 % 10 = 9.

Beispiel: Zur Ziffer an der Stelle 8 kommen Sie durch

  • "Abschneiden" einer Stelle: 123456789 / 10 = 12345678;
  • Ermitteln der letzten Ziffer: 12345678 % 10 = 8;

An die Ziffern der anderen Stellen kommen Sie auf analoge Weise.

Diese Funktion ist hier bereits vorgegeben. Sie brauchen Sie also nicht selber schreiben - verstehen sollten Sie sie aber schon. Finden Sie die Funktion in Aufgabe_05.java und verstehen Sie ihre Vorgehensweise.

Zahl und Prüfziffer zu ISBN-10 formatieren

Zuletzt bauen wir die oben geschriebenen Funktionen zu einer Gesamt-Funktion zusammen:

Schreiben Sie eine Funktion, die aus einer 9-stelligen Zahl z (z ist also der Parameter der Funktion) eine ISBN-10 als String zusammenbaut.

Der String soll in der Form „z1z2z3-z4z5z6z7-z8z9-P“ aufgebaut werden.

  • z1, z2, etc. entsprechen dabei den einzelnen Ziffern der Zahl z, wie wir sie oben bereits berechnet und genutzt haben
  • P entspricht dabei der Prüfziffer wie Sie sie in der Funktion in 2.3 berechnet haben. Geben Sie diesen String als Ergebnis zurück.

Beispiel: Wenn die Funktion mit der Zahl 386680192 als Parameter aufgerufen wird, wird der String "386-6801-92-0" zurückgegeben.

Hinweis: Strings werden mit dem Operator + konkateniert (d.h. zusammengebaut).

Implementieren Sie diese Funktion für eine formatierte ISBN-Ausgabe und testen Sie sie anschließend.
