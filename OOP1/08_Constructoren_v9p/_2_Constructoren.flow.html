Karten erzeugen

Nun sollen Karten-Objekte auch unter kontrollierten Bedingungen erzeugt werden können. 

Dazu definieren Sie nun einige Constructoren. Mit ihnen können Karten-Objekte erzeugt werden, mit und ohne explizite Angabe von initialen Daten. Dabei wird auch die neu hinzugekommenen Eigenschaft, Karten auch grafisch darstellen zu können, einbezogen.

Generierter Default-Constructor

Wenn in einer Klasse kein Constructor definiert ist, wird automatisch ein Default-Constructor vom Compiler zur Verfügung gestellt, der im Code benutzt werden kann ("Armenrecht").

Bisher haben Sie noch keinen Constructor in der Klasse Karte definiert. Daher überprüfen Sie als Erstes dieses "Armenrecht".

Was war nochmal der Default-Constructor?

Testen Sie, dass Sie ein Objekt der Klasse Karte unter Angabe des Default-Constructors erzeugen können.

Dokumentieren Sie im Testcode, dass das zu diesem Zeitpunkt funktioniert hat. 

 

Constructor mit Parametern 

Constructor mit Parametern definieren

Eine Karte ist sinnvollerweise nur zu verwenden, wenn folgende Daten sinnvolle Werte haben:

  • Position (Zeile, Spalte)
  • Aussehen (Text, Farbe)

Definieren Sie daher nun als erstes einen Constructor, bei dem alle diese Daten als Parameter übergeben werden - inclusive der Javadoc-Beschreibung.

Beim Erzeugen des Karten-Objekts muss nun auch ein Text-Objekt erzeugt und von der entsprechenden IV referenziert werden.

Und auch hier gilt:

Denken Sie daran, dass Sie auch im Constructor nicht einfach Werte in die IV schreiben.

Constructor verwenden

Wichtig: Da Sie nun zum ersten Mal etwas auf der Grafik-Konsole darstellen, müssen Sie vor dem Aufruf der Constructoren die Grafik-Konsole in main() initialisieren. Wie das geht, steht in der Beschreibung.

Nun erzeugen Sie zum Test drei Karten-Objekte an verschiedenen Positionen mit verschiedenem Aussehen und geben Sie dessen Eigenschaften auf die Konsole aus. Außerdem sollten diese Karten-Objekte nun auf der Grafik-Konsole zu sehen sein.

Wenn Sie keine oder nicht alle Text-Objekte auf der Grafik-Konsole sehen

 

Kein generierter Default-Constructor 

Da Sie nun selber einen eigenen Constructor definiert haben, wird Ihnen vom Compiler keiner mehr automatisch generiert.

Testen Sie nun, dass Sie kein Objekt der Klasse Karte unter Angabe des Default-Constructors erzeugen können.

Dokumentieren Sie am Testcode, dass das zu diesem Zeitpunkt nicht mehr funktioniert.

 

Standard-Karte - Default-Constructor

Default-Constructor definieren

Definieren Sie den Default-Constructor der Klasse Karte. 

Das ist nun sehr einfach, da Sie nun den oben bereits definierten Constructor aufrufen können. Es sollen folgende Standard-Werte verwendet werden:

  • Position = 0,0
  • Text = "Default", Farbe = "schwarz"
Beachten Sie die bekannten Randbedingungen für den Aufruf eines anderen Constructors der gleichen Klasse

Definieren Sie nun entsprechend den Default-Constructor- inclusive der Javadoc-Beschreibung.
