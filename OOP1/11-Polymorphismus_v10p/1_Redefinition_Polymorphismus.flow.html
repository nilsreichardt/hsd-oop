Karten sollen auf Interaktionen reagieren

Weiter im Spiel: Das Klicken auf eine Spiel-Karte soll eine spezielle Aktion für den Typ dieser Karte auslösen.

Beim Klick auf eine Karte wird die Methode klickAuf() für das Karten-Objekt aufgerufen. Das hatten Sie bereits im letzten Termin umgesetzt.

Damit für Objekte verschiedener Klassen verschiedene Dinge passieren, definiert man in jeder Subklasse von Karte die Methode klickAuf() auf eine andere Art und Weise. Damit wird die Methode in der Subklasse überschrieben (redefiniert).

Horizontale Karten tauschen mit dem Nachbarn

Reaktion von RKarte

Zuerst redefinieren Sie die Reaktion klickAuf() für die Klasse RKarte. Darin soll folgendes passieren:

  • Die Karte soll auf die Text-Konsole ausgegeben werden.
  • Die Karte tauscht mit dem rechten Nachbarn die Position.

Dabei verwenden Sie Methoden, die Sie bereits geschrieben haben. Die Methode sollte also sehr kurz und einfach aussehen. 

Beim Implementieren werden Sie feststellen, dass Sie der Compiler nicht auf das Spielfeld der RKarte zugreifen lässt. Warum ist das so?

Da die Subklassen von Karte eng mit der Karte verwandt sind, und sie alle gemeinsam erstellt werden, ist es daher zulässig, dass Sie die Sichtbarkeit dieser Instanz-Variablen so ändern, dass sie auch in Subklassen zugreifbar ist. 

Ändern Sie die Sichtbarkeit der IV - und nur dieser IV! Alle anderen sollen weiter private bleiben.

Machen Sie sich klar: 

  • Zur Laufzeit wird immer die Methode der Klasse des aktuellen Objekts aufgerufen.
  • Ohne, dass Sie den Code zum Aufruf der Methode klickAuf() geändert haben, wird unterschiedlicher Code ausgeführt - je nachdem, welcher Klasse die geklickte Karte angehört. 
  • Die Methoden heißen also alle gleich, sie führen nur unterschiedliche Aktionen durch.

Gegenbeispiel: keine Redefinition

Jetzt bauen Sie ein kleines Gegenbeispiel, in dem keine Redefinition und kein Polymorphismus vorliegt:

Definieren Sie eine Methode keineRedefintion() in jeder Subklasse von Karte, aber nicht (!) in Karte selber. Die Methode gibt nur "Test" auf die Text-Konsole aus.

Kopieren Sie den Code einfach in jede der vier Subklassen. Das muss nicht schön sein, wir wollen nur ein einfaches Beispiel aufbauen. 

Dann erweitern Sie main() so, dass nicht nur klickAuf() als Reaktion aufgerufen wird, sondern auch die gerade definierte Methode keineRedefintion().

Was meldet der Compiler, wenn Sie dann die Klasse FiveCrush compilieren?

Antwort: can not find symbol

 

Weil in der main von FiveCrush mit sucheKarte ein Karte-Objekt zurückgegben wird, welches die speziellen Methoden nicht kennt. 

Warum/Wann kann der Aufruf denn potentiell nicht funktionieren?

MusterLösung: Weil die Methode karteMitID() ein Karten-Objekt zurückgibt. D.h. ein Objekt des Typs Karte oder eines Subtyps von Karte. Aber eben auch von Karte. Und dort gibt es diese Methode nicht. Es reicht also nicht, dass Sie wissen, dass mit dem aktuellen Code kein Problem entstehen würde, wenn er so ausgeführt würde. Es muss immer klappen ("Schisserprinzip").

Kommentieren Sie nun den gerade eingefügten Aufruf in main() wieder aus, damit Sie weiterarbeiten können.

Punkte vergeben

Punkte werden unter anderem beim Klicken auf eine Karte vergeben. Dabei sollen folgende Regeln gelten:

  • Die "normalen" Karten O,U,R,L geben jeweils einen Punkt
  • Eine Farbeimer-Karte vereinfacht das Spiel, daher werden 3 Punkte abgezogen
  • Die Zufalls-Karte kann das Spiel vereinfachen, aber auch erschweren, daher gibt es 0 Punkte. 

Es gibt also eine Standard-Reaktion (1 Punkt vergeben) für alle Karten, sowie zwei spezielle Reaktionen.

Erweitern Sie daher die Methode klickAuf() in den passenden Klassen.

Beachten Sie, dass die Standard-Punktevergabe immer stattfindet

Musterlösung: Die passenden Klassen sind Karte (für die Standard-Reaktion), FKarte und ZKarte.

Nachbarn mit Wrapraound implementieren

Modifizieren Sie die Methode nachbarVon() so, dass ein "Wraparound" entsteht. D.h. der Nachbar einer Karte am linken Rand ist die Karte am rechten Rand und umgekehrt. Gleiches gilt für den oberen und unteren Rand. 

Damit können z.B. RKarten und LKLarten "im Kreis" geklickt werden und bleiben nicht am rechten Rand stehen. So kann man im Spiel mehr machen.
